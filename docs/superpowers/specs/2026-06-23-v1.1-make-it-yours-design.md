# Anota v1.1 "Make It Yours" - Design Spec

Date: 2026-06-23
Status: Approved for planning
Author: Adelson (with Claude)

## Summary

Anota v1.0 is live on the App Store as a free, private, no-ads Dominican domino
scorekeeper. v1.1 adds optional, privacy-preserving monetization through
cosmetic personalization, plus the infrastructure (over-the-air updates) that
makes every release after this one faster.

The release theme is "Make It Yours": the app stays fully free and fully
functional, and a single optional one-time purchase unlocks a set of premium
themes and theme-matched share cards, framed as supporting development.

No ads. No tracking. No reversal of the privacy positioning already shipped and
marketed in v1.0.

## Goals

1. Add a respectful, optional way for fans to pay, without ads and without
   breaking the "collects no data" story.
2. Make the app feel more premium and personal (themes).
3. Make the winning-share moment look great and spread the app (share card).
4. Ship over-the-air updates so future JS and theme changes skip App Review.
5. Establish a reusable theme + in-app-purchase pattern for apps #2 onward.

## Non-goals (explicitly out of scope)

- Ads of any kind, ever. The privacy story is the brand.
- Alternate app icons. Deferred to v1.2 because app icons are native and can
  never ship over-the-air, so they require their own build + review regardless.
  The theme system will be architected so adding them later is a small change.
- Custom build-your-own color picker. Deferred. The theme model will be designed
  so this is an additive follow-on if users ask for it.
- Any third-party analytics, advertising, or tracking SDK.
- RevenueCat or any third-party purchase server. Purchases use native billing
  only.

## Monetization model

- Single non-consumable in-app purchase: "Anota Pro".
- Price: $2.99 USD (Apple/Google equivalents in other regions). Price is set in
  the store and can change later with no app update.
- Unlocks: all premium themes + theme-matched share cards.
- Framed in the UI as unlocking the themes AND supporting development.
- Required "Restore Purchases" action.
- The app remains 100% functional for free. Only cosmetics are gated. This
  satisfies Apple/Google rules on free apps with optional purchases.

### Purchase technology

- Native billing via `react-native-iap` (StoreKit on iOS, Play Billing on
  Android). No third-party purchase server.
- One non-consumable product id, e.g. `dev.anota.pro`.
- Entitlement is validated and cached locally (AsyncStorage), refreshed by
  querying current entitlements / restoring on launch. For a $2.99 cosmetic
  unlock, on-device validation is appropriate; server-side receipt validation is
  unnecessary and would add a backend we explicitly do not want.
- This is a native dependency, so v1.1 requires a full EAS build + store review
  (expected and fine, since v1.1 is a build anyway).

## Theme engine

### Current state

[src/constants/colors.ts](../../../src/constants/colors.ts) exports a single
fixed `colors` object plus `teamPalette`. Components import `colors` directly.

### Target state

- Define a `Theme` type covering the existing token set: surfaces (bg, bgDeep,
  felt, hairline, divider), ivory tile tokens (tile, tileShadow, tileInk), text
  (text, textDim, textFaint), accent (gold), team colors (A, B with glow),
  danger, plus new fields: `id`, `name`, `tier` ('free' | 'pro'), `isLight`
  (drives status-bar style and any light-mode contrast handling), and an
  optional `iconName` reserved for v1.2 (unused in v1.1).
- A `ThemeProvider` / theme context exposes the active theme and a setter.
- Active theme id persists to AsyncStorage alongside the existing language and
  haptics preferences, and is restored behind the existing hydration gate.
- Components consume theme tokens from context (hook, e.g. `useTheme()`) instead
  of importing `colors` directly. This refactor is the bulk of the work.
- The legacy `colors` export is replaced by the Classic Felt theme so behavior
  is identical for free users.

### Themes (1 free + 5 Pro)

Starting palettes below match the approved mockups. Final hex are tuned during
implementation; where a team color collides with the theme accent, the team
color is nudged for distinctness while staying close to the mockup.

- Classic Felt (free): felt `#0F1F18`, bgDeep `#070D0A`, accent `#E6B449`, text
  `#F4EFE2`, team A `#E63946`, team B `#3B82F6`. Identical to live v1.0. Dark.
- Midnight (pro): navy felt `#0B1322`, accent `#6FA8FF`, text `#EAF1FB`, warm
  team `#FF7A6B`, cool team nudged to a teal for distinctness. Dark.
- Mahogany (pro): wood felt `#2B1A12`, accent `#D9A05B`, text `#F2E6D6`, team A
  `#E0734A`, team B `#7FB0C4`. Dark.
- Casino Red (pro): deep red felt `#2A0C0C`, accent `#E8C36B`, text `#F6E9DC`,
  team A `#F2C879`, team B `#E89B7A`. Dark.
- Bone (pro): light ivory bg `#EDE6D6`, accent `#B8862F`, ink text `#2A2117`,
  team A `#C0432B`, team B `#2F6DA8`. Light (drives dark status bar).
- Carbon (pro): near-black `#141414`, accent `#E6B449`, text `#F2F2F2`, team A
  `#FF6B5E`, team B `#5EA6FF`. Dark.

### Domino motif

- Each theme renders a subtle domino-tile watermark in the felt background:
  barely-there, low opacity, never over the score numbers. Purely decorative,
  preserves the one-glance legibility that is the app's core strength.
- The same domino-tile graphic is reused as the share card hero (see below).

## Theme picker UI

- New section in the existing Settings modal: a grid of theme swatches matching
  the mockup style (mini felt + accent + two team dots + name).
- Tapping a theme applies it instantly (live preview across the app).
- Pro themes show a small lock indicator until Anota Pro is unlocked. Tapping a
  locked theme opens the Pro sheet rather than applying.
- After unlock, all themes are selectable; the choice persists.

## Anota Pro sheet

- A simple sheet/modal: short value line ("Unlock every theme and support
  Anota"), the theme set preview, the price, a Buy button, and a Restore
  Purchases link.
- On successful purchase or restore, the entitlement is cached and the sheet
  dismisses to the now-unlocked picker.
- Handles the standard states: pending, success, cancelled, already-owned,
  error (with a friendly retry).

## Share card

- Replace the current plain-text share with a rendered image.
- Implementation: an off-screen styled component captured to an image via
  `react-native-view-shot`, then handed to the existing share flow. Native
  dependency (fits the v1.1 build).
- Card contents: themed felt background, gold "Anota" wordmark, final score
  (team names + points), the winner line, and a domino-tile hero showing the
  winning margin or capicúa.
- Free base card: always uses the Classic Felt look and always carries the
  "Anota" mark (this is free marketing; the mark stays on every shared image).
- Pro: the card matches the user's active theme.
- A plain-text fallback is retained for share targets that do not accept images.

## Over-the-air updates (expo-updates)

- Add `expo-updates`, configure the update URL via EAS, and set a
  `runtimeVersion` policy.
- After v1.1 ships, JS and asset changes (new themes, color tweaks, copy fixes,
  bug fixes) publish over-the-air in minutes with no App Review.
- Native changes (new libraries, SDK bumps, app icons) still require a build +
  review. This is the boundary that puts app icons in v1.2.

## Privacy impact

- The only change to the privacy posture is the optional purchase. Apple and
  Google process the transaction; that is inherent to any paid feature and is
  not data the app itself collects.
- No third-party SDK (no RevenueCat, no analytics, no ads), so no new data
  collectors are introduced.
- Update the hosted privacy policy (English + Spanish) with one honest line
  noting the optional in-app purchase is handled by the App Store / Play Store.
- App Store privacy nutrition label and Play Data Safety: review whether the
  optional purchase requires disclosure. If so, disclose only "Purchases" used
  for app functionality, not linked to identity, not used for tracking. The "no
  tracking, no ads" story stays intact.

## Versioning and housekeeping

- Bump app version 1.0.0 -> 1.1.0. Build number auto-increments via EAS.
- Refresh "What's New" copy (English + Spanish) for the store.
- Keep all copy free of em dashes (per standing preference).

## Testing

- Extend the existing Jest (RN-free) unit tests:
  - Theme persistence/selection reducer logic.
  - Pro entitlement gating logic (locked vs unlocked selection behavior), as a
    pure function independent of the native billing module.
- Manual device verification on iPhone SE (compact) and a large phone across all
  themes, including the light Bone theme (status bar, contrast), plus a real
  share-card capture and a sandbox purchase + restore.

## Build and release path

1. Implement behind the existing patterns (reducer + hook + persisted prefs).
2. `eas build --platform ios --profile production`, then `eas submit`.
3. Update the App Store listing "What's New"; submit for review.
4. Android follows once the Play account work is done (separate thread).

## Open questions

None blocking. Final theme hex and the exact domino-watermark opacity are tuned
during implementation against real devices.
